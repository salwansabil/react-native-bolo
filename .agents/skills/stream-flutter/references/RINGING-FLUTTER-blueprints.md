# Ringing - Blueprints

Load only the section you are implementing. For the SDK patterns, dashboard provider setup,
platform requirements, and gotchas behind these blueprints, see
[RINGING-FLUTTER.md](RINGING-FLUTTER.md). For client init and call basics, see
[VIDEO-FLUTTER.md](VIDEO-FLUTTER.md) / [VIDEO-FLUTTER-blueprints.md](VIDEO-FLUTTER-blueprints.md).

Blueprints in this file (verified against SDK v1.4.0 and the official ringing tutorial):

- **App entry + push-enabled init** - `main()`, Firebase, `StreamVideo` with push manager
- **Background FCM handler** - top-level isolate handler
- **Home screen wiring** - register observers for foreground / terminated incoming calls
- **Outgoing ring** - create and navigate to a ringing call
- **Call screen** - render the accepted call
- **iOS AppDelegate** - VoIP/CallKit registration
- **Android manifest + Gradle** - FCM permissions and plugin

## App Entry + Push-Enabled Init Blueprint

> **Docs:** [Push Notifications](https://getstream.io/video/docs/flutter/advanced/incoming-calls/push-notifications.md) · [Providers Configuration](https://getstream.io/video/docs/flutter/advanced/incoming-calls/providers-configuration.md)

```dart
// main.dart
import 'package:firebase_core/firebase_core.dart';
import 'package:flutter/material.dart';
import 'package:stream_video_flutter/stream_video_flutter.dart';
import 'package:stream_video_push_notification/stream_video_push_notification.dart';

import 'firebase_options.dart'; // generated by `flutterfire configure`

const apiKey = 'your_api_key';
const apnProviderName = 'your-apn-provider';     // matches Stream Dashboard
const fcmProviderName = 'your-fcm-provider';     // matches Stream Dashboard

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);

  final user = await loadStoredUser(); // null if not logged in
  if (user != null) {
    initStreamVideo(user); // construct the singleton before runApp if already logged in
  }

  runApp(const RingingApp());
}

StreamVideo initStreamVideo(StoredUser user) {
  return StreamVideo(
    apiKey,
    user: user.user,       // User.regular(userId:, name:, image:)
    userToken: user.token, // raw JWT string
    options: StreamVideoOptions(
      keepConnectionsAliveWhenInBackground: true,
    ),
    pushNotificationManagerProvider: StreamVideoPushNotificationManager.create(
      iosPushProvider: const StreamVideoPushProvider.apn(name: apnProviderName),
      androidPushProvider: const StreamVideoPushProvider.firebase(name: fcmProviderName),
      pushConfiguration: const StreamVideoPushConfiguration(
        ios: IOSPushConfiguration(iconName: 'IconMask'),
      ),
      registerApnDeviceToken: true, // iOS standard token for call.missed
    ),
  )..connect();
}
```

**Wiring:**

- `Firebase.initializeApp` is awaited **before** constructing `StreamVideo`
- The same `initStreamVideo` is called after login (see Home screen) and at startup when a
  stored user already exists
- `..connect()` is a cascade - device tokens register automatically once connected
- Persist `StoredUser` at login (e.g. `flutter_secure_storage`) so the background isolate
  can rebuild the client

## Background FCM Handler Blueprint

A **top-level** function (not a class method), annotated `@pragma('vm:entry-point')`, that
runs in its own isolate when a ringing FCM data message arrives in background/terminated
state.

> **Docs:** [Android / Firebase Integration](https://getstream.io/video/docs/flutter/advanced/incoming-calls/android-firebase-integration.md)

```dart
// push_background.dart
import 'package:firebase_core/firebase_core.dart';
import 'package:firebase_messaging/firebase_messaging.dart';
import 'package:stream_video_flutter/stream_video_flutter.dart';
import 'package:stream_video_push_notification/stream_video_push_notification.dart';

import 'firebase_options.dart';

@pragma('vm:entry-point')
Future<void> firebaseMessagingBackgroundHandler(RemoteMessage message) async {
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);

  final user = await loadStoredUser();
  if (user == null) return;

  // NON-singleton instance for this isolate.
  final streamVideo = StreamVideo.create(
    apiKey,
    user: user.user,
    userToken: user.token,
    pushNotificationManagerProvider: StreamVideoPushNotificationManager.create(
      iosPushProvider: const StreamVideoPushProvider.apn(name: apnProviderName),
      androidPushProvider: const StreamVideoPushProvider.firebase(name: fcmProviderName),
      registerApnDeviceToken: true,
    ),
  )..connect();

  final subscription = streamVideo.observeCoreRingingEventsForBackground();
  streamVideo.disposeAfterResolvingRinging(disposingCallback: subscription.cancel);
  await streamVideo.handleRingingFlowNotifications(message.data);
}
```

**Wiring:**

- Registered once with `FirebaseMessaging.onBackgroundMessage(firebaseMessagingBackgroundHandler)`
- `StreamVideo.create(...)` (not the singleton `StreamVideo(...)`) - the singleton does not
  exist in the background isolate
- `disposeAfterResolvingRinging` cancels the subscription and disposes the client shortly
  after the user accepts/declines or the call times out
- iOS background/terminated ringing is handled natively by CallKit/PushKit - this handler is
  primarily the Android path, but registering it is harmless cross-platform

## Home Screen Wiring Blueprint

Registers the foreground ringing observer and the Android terminated-state consumer, and
sets up FCM listeners.

> **Docs:** [Ringing](https://getstream.io/video/docs/flutter/advanced/incoming-calls/ringing.md) · [Android / Firebase Integration](https://getstream.io/video/docs/flutter/advanced/incoming-calls/android-firebase-integration.md)

```dart
// home_screen.dart
import 'package:firebase_messaging/firebase_messaging.dart';
import 'package:flutter/material.dart';
import 'package:rxdart/rxdart.dart';
import 'package:stream_video_flutter/stream_video_flutter.dart';

import 'call_screen.dart';
import 'push_background.dart';

class HomeScreen extends StatefulWidget {
  const HomeScreen({super.key});

  @override
  State<HomeScreen> createState() => _HomeScreenState();
}

class _HomeScreenState extends State<HomeScreen> {
  final _subscriptions = CompositeSubscription();

  @override
  void initState() {
    super.initState();
    FirebaseMessaging.instance.requestPermission();
    _tryConsumingTerminatedStateCall();
    _observeFcmMessages();
    _observeRingingEvents();
  }

  // Android only: a call accepted from the terminated state.
  void _tryConsumingTerminatedStateCall() {
    if (CurrentPlatform.isIos) return;
    WidgetsBinding.instance.addPostFrameCallback((_) {
      StreamVideo.instance.consumeAndAcceptActiveCall(
        onCallAccepted: _goToCall,
      );
    });
  }

  void _observeFcmMessages() {
    FirebaseMessaging.onBackgroundMessage(firebaseMessagingBackgroundHandler);
    _subscriptions.add(
      FirebaseMessaging.onMessage.listen((message) {
        StreamVideo.instance.handleRingingFlowNotifications(message.data);
      }),
    );
  }

  void _observeRingingEvents() {
    _subscriptions.add(
      StreamVideo.instance.observeCoreRingingEvents(
        onCallAccepted: _goToCall,
      ),
    );
  }

  void _goToCall(Call call) {
    Navigator.of(context).push(
      MaterialPageRoute(builder: (_) => CallScreen(call: call)),
    );
  }

  @override
  void dispose() {
    _subscriptions.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    // ... contact list + a "call" button that runs the outgoing-ring blueprint
    return const Placeholder();
  }
}
```

**Wiring:**

- `observeCoreRingingEvents` returns an rxdart `CompositeSubscription` (import
  `package:rxdart/rxdart.dart` — `stream_video` does not re-export it, and you must add
  `rxdart` to `pubspec.yaml`). Add it to a parent `CompositeSubscription` and call
  `dispose()` (or `clear()` to keep it reusable) in `dispose`.
- The same `_goToCall` is reused for foreground-accepted and terminated-accepted calls
- `consumeAndAcceptActiveCall` is guarded for Android - on iOS CallKit handles terminated
  accepts natively and pushes you the call through `observeCoreRingingEvents`

## Outgoing Ring Blueprint

> **Docs:** [Ringing](https://getstream.io/video/docs/flutter/advanced/incoming-calls/ringing.md)

```dart
import 'package:stream_video_flutter/stream_video_flutter.dart';
import 'package:uuid/uuid.dart';

Future<void> startRingingCall({
  required BuildContext context,
  required List<String> memberIds,
  bool video = true,
}) async {
  final call = StreamVideo.instance.makeCall(
    callType: StreamCallType.defaultType(),
    id: const Uuid().v4(), // unique per call
  );

  final result = await call.getOrCreate(
    memberIds: memberIds,
    ringing: true,
    video: video,
    // optional: ring: StreamRingSettings(autoCancelTimeout: Duration(seconds: 30)),
    // optional: custom: {'display_name': 'Team standup'}, // CallKit / notification title
  );

  result.fold(
    success: (_) {
      Navigator.of(context).push(
        MaterialPageRoute(builder: (_) => CallScreen(call: call)),
      );
    },
    failure: (failure) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text('Could not start call: ${failure.error.message}')),
      );
    },
  );
}
```

**Wiring:**

- `CallScreen` shows the outgoing ringing UI automatically while members are notified
- `memberIds` must be other users (never include yourself); they must already exist and have
  connected at least once
- Use `call.ring(userIds: [...])` instead to ring more people on an already-created call

## Call Screen Blueprint

`StreamCallContainer` renders the right screen based on the call status - outgoing ringing,
incoming ringing, or the active call - and provides accept/reject/leave/end controls.

> **Docs:** [Call Container](https://getstream.io/video/docs/flutter/call-container.md) · [Incoming Call](https://getstream.io/video/docs/flutter/incoming-call.md) · [Outgoing Call](https://getstream.io/video/docs/flutter/outgoing-call.md)

```dart
// call_screen.dart
import 'package:flutter/material.dart';
import 'package:stream_video_flutter/stream_video_flutter.dart';

class CallScreen extends StatelessWidget {
  const CallScreen({super.key, required this.call});

  final Call call;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: StreamCallContainer(
        call: call,
        // Customize ringing screens if needed:
        // incomingCallWidgetBuilder: (context, call) => MyIncomingScreen(call: call),
        // outgoingCallWidgetBuilder: (context, call) => MyOutgoingScreen(call: call),
        onCallDisconnected: (_) {
          if (Navigator.of(context).canPop()) Navigator.of(context).pop();
        },
      ),
    );
  }
}
```

**Wiring:**

- One widget handles the full ringing lifecycle; no manual accept/reject wiring needed
- `onCallDisconnected` pops back when the call ends (declined, cancelled, or hung up)
- For manual control, call `call.accept()` / `call.reject(reason: CallRejectReason.decline())`
  / `call.end()` yourself and build custom screens via the `*WidgetBuilder` params

## iOS AppDelegate Blueprint

> **Docs:** [iOS / CallKit Integration](https://getstream.io/video/docs/flutter/advanced/incoming-calls/ios-callkit-integration.md) (docs show the older `GeneratedPluginRegistrant.register(with: self)` AppDelegate; the `FlutterImplicitEngineDelegate` template below is the current Flutter 3.35+ form — keep it).

```swift
// ios/Runner/AppDelegate.swift
import Flutter
import UIKit
import stream_video_push_notification

@main
@objc class AppDelegate: FlutterAppDelegate, FlutterImplicitEngineDelegate {
  override func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
  ) -> Bool {
    // Registers PushKit/VoIP and reports incoming calls to CallKit.
    StreamVideoPKDelegateManager.shared.registerForPushNotifications()

    if #available(iOS 10.0, *) {
      UNUserNotificationCenter.current().delegate = self
    }

    return super.application(application, didFinishLaunchingWithOptions: launchOptions)
  }

  // Plugin registration moved here in the implicit-engine template — leave it intact.
  func didInitializeImplicitFlutterEngine(_ engineBridge: FlutterImplicitEngineBridge) {
    GeneratedPluginRegistrant.register(with: engineBridge.pluginRegistry)
  }
}
```

Required `ios/Runner/Info.plist` entries:

```xml
<key>UIBackgroundModes</key>
<array>
  <string>audio</string>
  <string>processing</string>
  <string>remote-notification</string>
  <string>voip</string>
</array>
<key>NSCameraUsageDescription</key>
<string>Calls require camera access.</string>
<key>NSMicrophoneUsageDescription</key>
<string>Calls require microphone access.</string>
```

**Wiring:**

- The single `registerForPushNotifications()` call is the entire VoIP/CallKit wiring - the
  plugin owns `PKPushRegistryDelegate` and CallKit reporting
- Add the **Push Notifications** capability in Xcode (creates `aps-environment`) and enable
  the Background Modes above - without the capability iOS silently drops pushes
- Deployment target must be iOS 14.0+

## Android Manifest + Gradle Blueprint

> **Docs:** [Android / Firebase Integration](https://getstream.io/video/docs/flutter/advanced/incoming-calls/android-firebase-integration.md)

`android/app/src/main/AndroidManifest.xml`:

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <uses-permission android:name="android.permission.INTERNET"/>
    <uses-permission android:name="android.permission.CAMERA"/>
    <uses-permission android:name="android.permission.RECORD_AUDIO"/>
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>
    <uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS"/>
    <uses-permission android:name="android.permission.BLUETOOTH" android:maxSdkVersion="30"/>
    <uses-permission android:name="android.permission.BLUETOOTH_ADMIN" android:maxSdkVersion="30"/>
    <uses-permission android:name="android.permission.BLUETOOTH_CONNECT"/>
    <uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>

    <application android:label="Ringing App" android:icon="@mipmap/ic_launcher">
        <activity
            android:name=".MainActivity"
            android:launchMode="singleInstance"
            android:exported="true">
            <!-- ... -->
        </activity>
    </application>
</manifest>
```

`android/app/build.gradle` (apply the Google Services plugin):

```groovy
plugins {
    id "com.android.application"
    id "com.google.gms.google-services" // FlutterFire / FCM
    id "kotlin-android"
    id "dev.flutter.flutter-gradle-plugin"
}
```

`android/settings.gradle` (declare versions):

```groovy
plugins {
    id "com.android.application" version "8.12.1" apply false
    id "com.google.gms.google-services" version "4.3.15" apply false
    id "org.jetbrains.kotlin.android" version "2.2.0" apply false
}
```

`MainActivity.kt` stays a plain `FlutterActivity`:

```kotlin
package com.example.your_app

import io.flutter.embedding.android.FlutterActivity

class MainActivity : FlutterActivity()
```

**Wiring:**

- Place `google-services.json` at `android/app/`
- `launchMode="singleInstance"` (or `singleTask`) keeps notification taps from stacking call
  screens
- Request `POST_NOTIFICATIONS` at runtime on Android 13+ (`permission_handler`); call
  `StreamVideoPushNotificationManager.ensureFullScreenIntentPermission()` on Android 14+ for
  lock-screen full-screen ringing
- Foreground-service permissions are pulled in transitively by the plugin - you only add
  `POST_NOTIFICATIONS`
